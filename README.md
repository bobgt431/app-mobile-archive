# app-mobile-archive

## 项目简介

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面通常为移动端适配的静态内容，不绑定任何特定的域名或网站。仓库的主要目的是提供一个结构化的存储空间，方便对多个独立页面进行版本管理、历史追溯和静态托管。

## 目录说明

```
app-mobile-archive/
├── pages/          # 存放所有归档的 HTML 页面文件
├── assets/         # 页面引用的静态资源（css、js、图片等）
├── index.html      # 页面列表索引（可选）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件代表一个独立的归档页面，文件名建议使用有意义的标识（如 `page-001.html`）。  
- `assets/`：按需存放页面依赖的资源文件，建议按类型或页面分组。  
- `index.html`：可选的导航页，用于快速访问所有归档页面（非必须）。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，可直接在浏览器中打开或通过静态托管服务访问。  
- 页面内容通常为移动端优化视图，但不限制具体表现形式或用途。  
- 归档的页面可能包含多种主题、功能或演示用途，不针对单一领域。  
- 每个页面应尽量自包含，减少对外部资源的依赖（如有依赖，建议一并存入 `assets/` 目录）。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，如有资源文件同步放入 `assets/`，并在需要时更新 `index.html`。  
- 更新页面：直接修改对应 HTML 文件，建议在提交信息中简要说明变更内容。  
- 删除页面：移除对应文件及引用资源，避免残留链接。  
- 不建议在页面内添加外部追踪脚本、广告或动态跳转链接，保持归档的纯粹性。

## 许可

本项目未指定特定许可证，仅作为页面归档用途。各页面内容版权归原始作者所有。
